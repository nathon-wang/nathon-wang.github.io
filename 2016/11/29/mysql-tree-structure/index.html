使用关系型数据库表示树形结构

发表于 2016-11-29 |

树形结构是很自然、常用的一种数据结构，比如公司的组织架构、目录树、某些分类系统。但是我们目前的数据系统大多是关系型的与树形结构天然异构，所以用关系型数据库存储树状结构往往很复杂。
要想在关系数据库中存树形结构主要有以下四种方案
邻接表(Adjacency list)
路径枚举(Path enumeration)
嵌套集合(Nested sets)
闭包表(Closure table)

以以下树形结构为例，说下这四种树形结构存储方式的利弊

先考虑实际使用场景，常见的树形结构的操作有以下7种

1.给定节点求直接子节点
2.给定节点求其父节点
3.给定节点求节点及其节点子树
4.给定节点求节点到根的路径
5.移动节点及所在的子树
6.删除节点及所在子树
7.插入一个新节点

操作6跟操作3实际是相同的，只不过一个是SELECT，一个是DELETE。操作5移动节点，实际是选取节点及子树进行删除，然后按照规则在目标节点下进行INSERT的复合操作，也就是6+7。
所以实际就是下面5中操作

1.给定节点求直接子节点
2.给定节点求其父节点
3.给定节点求节点子树
4.给定节点求节点到根的路径
5.插入一个新节点

邻接表

邻接表是最简单的表示树形结构的方法，因为子节点与其父节点是一对一的关系，邻接表在节点表中存储每个节点的父节点。当然这种做法的问题就是，如果你想求一个节点的子节点，相当麻烦。。。

CREATE TABLE Node_al (
id INT(10) NOT NULL,
name VARCHAR(10) NOT NULL,
parent_id INT(10) NOT NULL,
PRIMARY KEY(id)
);

INSERT INTO Node_al VALUES (1, ‘N1’, 0), (2, ‘N2’, 1), (3, ‘N3’, 1), (4, ‘N4’, 2), (5, ‘N5’, 3), (6, ‘N6’, 3), (7, ‘N7’, 4), (8, ‘N8’, 4);

路径枚举

路径枚举指的是将节点的路径作为一个字段存到数据表中，比如下面这样
CREATE TABLE Node_pe (
id INT(10) NOT NULL,
name VARCHAR(10) NOT NULL,
path VARCHAR(128) NOT NULL,
PRIMARY KEY(id)
);

INSERT INTO Node_pe VALUES (1, ‘N1’, ‘/1’), (2, ‘N2’, ‘/1/2’), (3, ‘N3’, ‘/1/3’), (4, ‘N4’, ‘/1/2/4’), (5, ‘N5’, ‘/1/2/5’), (6, ‘N6’, ‘/1/3/6’), (7, ‘N7’, ‘/1/3/7’), (8, ‘N8’, ‘/1/2/4/8’);
最后的表如下图

这样做的好处就是无需计算一个节点所在的路径，因为已经存在表中了

考察以上5种操作

求节点2的所有直接子节点
SELECT * FROM Node WHERE REGEXP ‘^[0-9/]/2/[0-9]+/$’;

求节点2的父节点
SELECT n1.* FROM Node n1 JOIN Node n2 ON n1.id = n2.parent_id WHERE n2.id = 2;

求节点2的所有子孙节点
求一个节点的父节点需要内连接一次，我们要从叶子节点求其父节点，然后求其父节点的父节点，直到满足条件node_id = 2。有多少层就要连接多少次，所以如果你有很多求子树的操作，而且树很深，此方式不适用

求节点8到根的路径
SELECT path FROM Node WHERE node_id = 8;

移动节点4及其所在的子树到节点5
因为一个节点只与上层的父节点关联所以移动节点非常简单，只需改变需要移动的子树的根
UPDATE Node SET parent_id = 5 WHERE node_id = 4;

删除节点及所在子树
同求节点的子孙节点，需要锁定读之后执行删除操作

在节点5上插入节点9
INSERT INTO Node VALUES (9, ‘Node9’, 5);

嵌套集

嵌套集合将树形结构想象成一系列的集合，比如下面这样

id name left right
1 N1 1 16
2 N2 2 9
3 N3 10 15
4 N4 3 8
5 N5 11 12
6 N6 13 14
7 N7 4 5
8 N8 6 7

考察以上7种操作

求节点2的所有直接子节点
SELECT * FROM Node AS current, Node AS parent WHERE current.left > parent.left AND current.right < parent.right AND current.id = 2;

求节点2的父节点
SELECT * FROM Node AS current, Node AS parent WHERE current.left > parent.left AND current.node_id = 2 ORDER BY parent.left DESC LIMIT 1;

求节点2的所有子孙节点
